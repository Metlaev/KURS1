## Выполненные задания

### Задание 1. Создание потоков
**Описание:** Написана программа, создающая дочерний поток с атрибутами по умолчанию (`NULL`). Родительский и дочерний потоки выводят в консоль по 5 строк текста в произвольном порядке.

<details>
<summary>Исходный код (Задание 1)</summary>

```c
#include <pthread.h>
#include <stdio.h>
#include <unistd.h>

#define N 4

void *routine(void *_){
    for (int i=0; i < N; i++){
        printf("thread: i=%d\n", i);
        sleep(2);
    }
    return NULL;
}
int main(void) {
    pthread_t tid;
    pthread_create(&tid, NULL, routine, NULL); // атрибуты по умолчанию
    pthread_join(tid, NULL); // переместил сюда, чтобы родительский поток выводил \
                                текст после завершения дочернего
    for (int i=0; i < N; i++){
        printf("main procces: i=%d\n", i);
        sleep(1);
    }
    //pthread_join(tid, NULL) был здесь до 2 задания
}
```
</details>

### Задание 2. Ожидание завершения потока
**Описание:** Модифицирована программа из Задания 1. Вызов функции `pthread_join()` перенесен в коде выше — сразу после вызова `pthread_create()`. 

**Результат:** Родительский поток блокируется и ожидает полного завершения дочернего потока. В консоли строго сначала выводятся все 5 строк дочернего потока, и только затем — 5 строк родительского.

### Задания 4 и 5. Принудительное прерывание потоков и обработка завершения
**Описание:** Создаются дочерние потоки, выполняющие бесконечный цикл со спячкой в `sleep(1)`. Главный поток засыпает на определенное время, после чего вызывает `pthread_cancel()` для каждого потока. 
Для предотвращения утечек ресурсов внутри потоков зарегистрирована функция очистки через макросы `pthread_cleanup_push()` и `pthread_cleanup_pop()`. При отмене потока автоматически срабатывает обработчик очистки.
<details>
<summary>Исходный код (Задания 3, 4, 5)</summary>

```c
#include <pthread.h>
#include <stdio.h>
#include <unistd.h>

#define N 4

typedef struct {
    int id;
    char **lines;
}ThreadData;

// функция для 5 задания
void message(void *arg){
    int id = *(int*)arg; // разыменовываем указатель с явным приведением к int
    printf("Завершение потока %d\n", id);
}

void *print_seq(void *arg){
    ThreadData *data = (ThreadData*)arg;
    int id = *(int*)arg;
    pthread_cleanup_push(message, &id); // 5 задание: зарегали ф-ию message в стек обработчиков
    for (int i=0; i < 3; i++){
        printf("Поток %d: %s\n", data->id, data->lines[i]);
        sleep(1); // 4 задание: сон на 1 сек
    }
    pthread_cleanup_pop(0); // удаляет из стека ф-ию
    return NULL;
}
int main(void) {
    pthread_t tid[4];
    ThreadData data[4];
    char *seq[4][3] = {
        {"1", "2", "3"},
        {"a", "b", "c"},
        {"яблоко", "груша", "банан"},
        {"лев", "тигр", "гепард"}
    };
    for (int i=0; i < 4; i++){
        data[i].id = i + 1;
        data[i].lines = seq[i];
        pthread_create(&tid[i], NULL, print_seq, &data[i]);
    }
    sleep(2); // 4 задание: перед прерыванием сон на 2 секунды
    for (int i = 0; i < 4; i++) {
        pthread_cancel(tid[i]); // 4 задание: прерываем работу всех дочек
        // после отмены срабатывает функция из pthread_cleanup_push
        pthread_join(tid[i], NULL);
    }
}

```
</details>

### Задание 6. Реализация алгоритма Sleepsort
**Описание:** Реализован алгоритм сортировки массива чисел с помощью потоков. Для каждого элемента массива порождается отдельный поток, который засыпает на время, пропорциональное значению этого элемента (`usleep(val * 10000)`). Потоки с меньшими значениями просыпаются раньше и первыми выводят числа на экран, формируя отсортированный массив.
